## 项目规范
> 如遇到不符合规范的情况，都会责令修改！

> ！！！！！以下规范非常重要，一定查阅！！！！
```text
1. 逻辑开发使用Part组件
    1)怎么划分职责：每个人负责自己的组件，不能修改别人负责的组件
    2)如何协作：开发组件的同学负责接口和事件分发
    3)如果一定要改怎么做：在群里@一灯 @卡尔，修改内容
2. 视图逻辑使用BaseView继承类，UIManager的继承类中声明
3. 大厅严禁引用子游戏代码
4. Core包严禁引入Game包代码
5. 事件名称必须要有声明
6. 不能在代码中硬编码常量数值或者有明确业务意义的字符串（通过配置化解决，详情见“框架.md”）
7. 严格按照规范提交代码
```
## 编码规范
```text
1.单一功能原则：一个函数只干一件事，一个文件或者一个类必须只做一个模块的事情（比如登录，不要去计算金币等）
2.强内聚，解耦合
3.代码尽量干净整洁，列数不能超过一屏幕
4.单文件不能超过400行
5.使用PascalCase为类型命名
6.使用PascalCase为枚举值命名
7.使用camelCase为函数命名
8.使用camelCase为属性或本地变量命名
9.私有用"_"前缀区分，一定要加private
10.尽可能使用完整的单词拼写命名
11.函数与函数之间只能空一行
12.一个文件里面只能有一个类
13.按照目录结构存放文件
14.Part组件代码不耦合其他Part组件，通过事件解耦合；如需使用，需要检查Part是否存在
```
## 目录结构
```text
Script
├──Core 框架核心包
├────Enum 枚举类
├────Manager 管理类
├────Network 网络包
├────UI UI框架
├──Util 工具
├──Game 业务包
├────AssetManager 热更新模块
├────Config 配置解析类
├────Const 常量类
├────DataStruct 数据结构
├────Entities 实体（少用）
├────Enum 枚举类
├────ErrorCode 服务器异常编码
├────Event 事件常量定义
├────Logs 日志上传/收集工具
├────Manager 管理器，比如UI管理器，图集管理
├────Parts 所有逻辑组件
├────Scene 场景
├────Test 测试用例
├────UI 视图类
├────Utils 工具类
├────World 状态类
├──Lib 第三方ts/js库
├──Utils 框架工具
```
